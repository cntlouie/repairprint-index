# Security reporting

This bootstrap is not a public production service. Before launch, replace this file with a monitored private security contact and response policy.

Do not file public issues containing:

- Authentication bypass details
- Database/storage/service credentials
- Private model-label or fitment photos
- Email addresses, serial numbers, receipts, or other contributor data
- Exploitable arbitrary-URL-fetch behavior

Production must have a private reporting channel, named responder, severity model, acknowledgement target, emergency unpublish path, and incident-retention policy.

High-severity RepairPrint issues include wrong-model recommendations that could cause harm, public exposure of private submission media, blocked safety parts becoming public, source-adapter credential compromise, and rights/safety notices failing to place a record on hold.
